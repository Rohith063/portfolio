🎙️ Voice AI · Chess Engine · Full-Stack

Voice Chess AI — Play Chess with Your Voice

What is this?

A fully working chess application where you speak your move and Stockfish — one of the strongest chess engines in the world — responds and announces its move aloud. The app handles voice input via Whisper, processes natural language moves via a custom NLP parser, validates all moves using python-chess, and streams game state over WebSockets.

Features

Voice input via Whisper STT
Voice output via pyttsx3 TTS
Click-to-move board UI
Natural language move parser
Full legal move validation
Undo last two half-moves
Real-time WebSocket play
Stockfish strength settings

How I built it

  1. 1Built FastAPI backend with WebSocket game loop — real-time bidirectional communication between browser and chess engine
  2. 2Integrated OpenAI Whisper for offline speech-to-text transcription of voice moves
  3. 3Wrote a custom NLP move parser that interprets natural phrases like "knight to f3", "castle kingside", "pawn takes d5" into UCI notation
  4. 4Wrapped Stockfish as a subprocess — adjustable skill level (1–20) and depth for difficulty control
  5. 5Added pyttsx3 (offline) / gTTS (online) TTS so Stockfish announces its moves aloud
  6. 6Built single-file HTML frontend with interactive board, piece click-to-move, highlights and status display

What makes this stand out

Most student chess projects are either a simple board UI or a basic Stockfish wrapper. This one is a complete voice-first AI gaming experience — combining Whisper (state-of-the-art STT), Stockfish (world-class engine), custom NLP parsing, and real-time WebSocket communication in a single deployable Python app.

🤖 ML + Graph Theory · Solo Research

Hybrid Graph-Theoretic & ML Framework for Hyderabad Metro Route Optimization & Fare Prediction

Problem Statement

Hyderabad Metro Rail connects 57 stations across 3 corridors, but commuters often can't find the most efficient route or predict their fare before travelling. No unified intelligent tool existed that solved both. I built one — solo, from scratch, applied to real Hyderabad Metro data.

Approach

  1. 1Modelled the entire Hyderabad Metro network as a weighted graph in Python using NetworkX — stations as nodes, routes as edges weighted by distance and time
  2. 2Implemented Dijkstra's Algorithm to compute the shortest efficient path between any two stations, including interchange handling
  3. 3Engineered fare features (distance, zones, peak/off-peak, interchange count) and trained a Decision Tree Regressor with Scikit-learn
  4. 4Integrated graph engine + ML model into a single Flask web app — one input, both outputs
  5. 5Visualized the metro network and highlighted routes using Matplotlib

Results & Impact

A working end-to-end system combining classical DSA and modern ML — deployed as a Flask web application applied to real Hyderabad Metro data. One of the few student-built tools on an Indian urban transit system. Demonstrates the ability to model real infrastructure, not just run a tutorial dataset.
