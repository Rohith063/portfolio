🤖 ML + Graph Theory · Solo Research Project

Hybrid Graph-Theoretic & ML Framework for Hyderabad Metro Route Optimization & Fare Prediction

Problem Statement

Hyderabad Metro Rail connects 57 stations across 3 corridors, but commuters often can't identify the most efficient route or accurately predict their fare before travelling. There was no unified, intelligent tool that combined optimal pathfinding with dynamic fare prediction. I set out to build one — from scratch, solo.

Approach

  1. 1Modelled the entire Hyderabad Metro network as a weighted graph using Python and NetworkX — stations as nodes, routes as edges weighted by distance and travel time
  2. 2Implemented Dijkstra's Algorithm to compute the shortest and most efficient path between any two stations, handling interchanges and multi-corridor journeys
  3. 3Collected and processed metro fare data, engineered features (distance, zones, peak/off-peak, interchange count) and trained a Decision Tree Regressor using Scikit-learn for accurate fare prediction
  4. 4Integrated both the graph engine and the ML model into a unified Flask web application — users input source and destination, get back the optimal route AND the predicted fare in one response
  5. 5Visualized the network and route highlights using Matplotlib for both analysis and the web interface

Results & Impact

Successfully built a working end-to-end system that correctly optimizes routes across the Hyderabad Metro network using Dijkstra's algorithm and predicts fares using a Decision Tree Regressor — both integrated into a single Flask web application. This project demonstrates the practical power of combining classical DSA with modern ML, and represents one of the few student-built tools applied to a real Indian urban transit system.

What Makes This Stand Out

Most ML projects at this level use standard datasets and train a model. This project required modelling a real-world city infrastructure as a graph, implementing a pathfinding algorithm from scratch, training a regression model on domain-specific data, and wiring everything together into a deployed web app — all solo.

🌐 Web App · Front-End Development

4Ever — Relationship Planning & Budget Tracker

Problem Statement

Couples often juggle shared finances, date planning and day-to-day scheduling across multiple apps or not at all. The idea behind 4Ever was simple: build a single, focused web app designed specifically for two people — where they can track their shared budget, plan their days together and manage their relationship in one place.

Approach

  1. 1Defined the core feature set: shared budget tracker, day-to-day scheduler, and relationship milestone planning — keeping scope focused and user-centred
  2. 2Designed a clean, warm UI using HTML and CSS — intuitive enough for non-technical users, visually distinct from generic productivity apps
  3. 3Built interactive features in vanilla JavaScript — budget calculations, schedule entries, persistent local state using browser storage
  4. 4Ensured the app was fully responsive for mobile use, since most couples would access it from their phones

Results & Impact

A fully working, deployed front-end web application with budget planning, day scheduling and relationship features built entirely in HTML, CSS and JavaScript. This project demonstrates UI/UX thinking, front-end development skills and the ability to scope, design and ship a complete product independently.

What I Learned

Building for a human need (not just a technical challenge) forced me to think about user experience first. This project sharpened my HTML/CSS skills, my JavaScript DOM manipulation and my ability to design something that feels genuinely usable — not just functional.
